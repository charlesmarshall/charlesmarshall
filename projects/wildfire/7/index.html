Wildfire CMS version 7

While working on a couple of larger projects that required tighter integration with multi-media channels such as youtube and flickr.

As the CMS already had a basic file system to allow local uploads and render from files saved there it made sense to give this an update.

A big change to the how media works under the hood, so any sites running on older versions need some time to upgrade, but otherwise it all looks the same.

Abstraction

Instead of just giving the file system a quick update, it was time to make it more robust and more flexible. Uploading files to one location is fine, but when you have dozens of videos stretched out accross youtube, vimeo and a few other places you really don't want the pain of having to re-upload them.

In a similar vain, storing, transcoding and having cutting edge video playback tech for all of those videos isn't exactly cost effective when you're already paying for Pro accounts with other providers.

With that in mind it made sense to utilise these existing platforms instead of trying to remake the wheel, especially as most have APIs

Standing on shoulders

It made a lot of sense to make use providers such as youtube and vimeo instead of trying to create our own. They have teams of people keeping their players up to date with the latest standards and platforms so we don't have to.

Making a media object which stores some very basic information to uniquely identifiy itself and what kind of media it was (ie youtube / vimeo etc) the standard content to files join is maintained and is simple enough to upgrade from v6.pretty style to this method.

Keeping it simple

The way the code accesses and uses the media changes slightly, but kept it nice and simple with all work being done for you.

As the media knows what kind of item it is, it handles what it returns. It creates the right classes and calls the right methods so you end up with an img, iframe or whatever is suitable.

Not all sites need this kind of complexity, so we've kept the standard file upload to server style in, but 3rd party based media is split to their own respective plugins.

Most 3rd party APIs require OAUTH or similar to make use of them, but there are files included to help sort that.

Go fetch

Once the website is authorised and configured, you of course want to sync your videos / images from that source, so that is built in as well.

As part of the new media section you can use the sync tool to update the CMS; you just need to click the button and follow the wizard. It will fetch all the information it needs, tag them and add them to your media library.
