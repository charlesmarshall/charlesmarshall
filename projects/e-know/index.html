Animations

A big part of this build were the header & data centre animations. As we had to support old versions of IE this meant we had to use javascriot in order to create these effects.

By keeping all images the same size as the canvas area this allowed for a simplified animation where everything is in the correct place by defauly and you just have to move images ot the new locations. This kept the css and transitions much cleaner, very is very handy considering how many there are.

Each animation stage is triggered within custom jquery event, which itself is called form a bound event on a pagination link.

You can see an isolated example of the animation working on my demo page

Content entry templates

To spice up the layout of the page content we created several alternative layouts that can be selected within the editor.

This is a built in feature for the last couple of versions of Wildfire CMS, it makes use of TinyMCEs function, but we overwrite it with a dynamic listing created from the CMS.
